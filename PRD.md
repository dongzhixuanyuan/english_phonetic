# 小学生英语课本音标点读 APP - 产品需求文档（PRD）

## 版本信息

| 项目 | 内容 |
|------|------|
| 文档版本 | v1.0 |
| 创建日期 | 2026-04-14 |
| 产品定位 | 个人定制向通用工具演进的小学生英语课本音标点读 APP |
| 目标平台 | iOS（iPhone / iPad） |
| 核心用户 | 小学生家长 |
| 终端用户 | 小学生 |

---

## 1. 产品背景与目标

### 1.1 背景

- 小学英语课时少、班级人数多，老师分配到每个孩子身上的时间有限
- 家庭缺乏英语练习环境，家长辅导发音能力普遍不足
- 学校教授自然拼读，但孩子课后缺乏配套资源和环境巩固
- 孩子普遍存在"学过的单词不会读"的问题

### 1.2 产品目标

打造一款家长可以轻松上手的工具：通过拍照课本，APP 自动识别单词并标注音标，点击即可发音。让孩子在没有家长辅导的情况下，也能独立、准确地朗读课本单词。

### 1.3 产品 Slogan

> **给课本拍张照，秒变会说话的音标点读机。**

---

## 2. 用户故事

### 2.1 核心用户故事

**作为家长**，我希望把孩子的英语课本拍照上传到 APP，这样孩子点击单词就能听到标准发音，不用我亲自辅导。

**作为家长**，我希望自动识别的音标位置可以手动调整，这样即使 OCR 有偏差，我也能快速校对准确。

**作为孩子**，我希望打开 APP 就能直接看到熟悉的课本页面，点击音标就能听到发音，操作简单不复杂。

## 4. 功能需求

### 4.1 首页

#### 4.1.1 页面结构

| 区域 | 内容 | 优先级 |
|------|------|--------|
| 顶部 | APP Logo + Slogan | P1 |
| 中部 | 示例页卡片（3-5 张可横向滑动） | P1 |
| 下部 | "我的课本" 列表入口 | P1 |
| 底部 | 底部导航栏（首页 / 设置） | P2 |

#### 4.1.2 示例页

- 预置 3-5 页精校过的教材页面
- 点击卡片直接进入点读模式
- 目的：让用户无需任何操作即可体验核心功能
- 示例页需覆盖常见版式：单词列表页、对话页、图文混排页

#### 4.1.3 我的课本列表

- 以卡片形式展示用户已添加的课本
- 卡片显示：课本封面（第一页缩略图）、课本名称、总页数、最近学习时间
- 点击卡片进入课本详情页
- 右上角 "+" 按钮添加新课本

---

### 4.2 课本管理

#### 4.2.1 添加课本

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| 课本名称 | 如"人教版 PEP 三年级上册" | 是 |
| 课本封面 | 自动使用第一页图片作为封面 | 否 |

- 添加课本后自动进入"添加页面"流程

#### 4.2.2 课本详情页

- 顶部显示课本封面和名称
- 下方为单元列表，默认展开第一个单元
- 每个单元显示包含的页面缩略图
- 支持单元折叠/展开
- 支持长按页面缩略图进行删除、重命名

#### 4.2.3 添加页面

1. **选择图片来源**
   - 拍照
   - 从相册选择

2. **图片预处理（可选）**
   - 基础裁剪：自由裁剪图片边缘
   - 透视校正：四边形拖拽校正拍摄变形（P2）

3. **确认导入**
   - 选择所属单元（可新建单元）
   - 输入页面名称（如"Page 2"，可选，默认自动编号）

4. **自动处理**
   - 进入识别中状态，显示进度提示
   - 完成后自动进入校对模式

---

### 4.3 核心功能：点读页

#### 4.3.1 页面布局

| 元素 | 说明 |
|------|------|
| 课本图片 | 占满屏幕主体区域，支持双指缩放、拖拽 |
| 音标标签 | 叠加在图片上，显示在对应单词附近 |
| 播放指示器 | 点击发音时，对应标签高亮或脉冲动画 |
| 工具栏 | 底部悬浮工具栏，提供模式切换 |

#### 4.3.2 两种模式

| 模式 | 说明 | 操作 |
|------|------|------|
| **点读模式** | 默认模式，点击音标/单词播放发音 | 单击标签播放 TTS |
| **校对模式** | 编辑模式，可调整识别结果和位置 | 长按标签进入编辑菜单，或从工具栏切换 |

#### 4.3.3 音标标签设计

- 标签背景：白色半透明圆角矩形
- 文字颜色：深灰色（音标）+ 浅灰色小字（原单词，可选显示）
- 字体：系统字体为主，IPA 特殊字符使用嵌入音标字体
- 标签大小：根据音标长度自适应
- 点击反馈：轻微缩放 + 颜色变化

---

### 4.4 校对模式

#### 4.4.1 进入方式

- 点读页工具栏点击"编辑"按钮
- 首次自动识别完成后自动进入

#### 4.4.2 校对功能清单

| 功能 | 操作方式 | 优先级 |
|------|----------|--------|
| **修改识别文字** | 点击标签 → 弹出编辑框，修改单词文本 | P1 |
| **拖拽调整位置** | 长按标签 → 拖拽到正确位置 | P1 |
| **删除多余标签** | 长按标签 → 菜单选择"删除" | P1 |
| **手动添加标签** | 点击图片空白处 → 输入单词 → 生成音标标签 | P1 |
| **重新识别** | 工具栏"重识"按钮，清空当前标注重新 OCR | P2 |
| **撤销/重做** | 工具栏提供撤销/重做按钮 | P2 |

#### 4.4.3 修改单词后的联动

- 修改单词文本后，自动重新查询本地词库获取音标
- 若词库中不存在，提示用户手动输入音标（或留空仅显示单词）
- 手动输入的音标自动保存到本地词库

---

### 4.5 OCR 与音标系统

#### 4.5.1 OCR 技术方案

- **框架**：Apple Vision (`VNRecognizeTextRequest`)
- **配置**：
  - `recognitionLevel = .accurate`
  - `recognitionLanguages = ["en-US"]`
- **输出**：单词文本 + 归一化 boundingBox（x, y, width, height）
- **后处理**：
  - 过滤过短字符（< 2 个字母，可选配置）
  - 过滤纯数字、纯符号
  - 合并同一行内过近的识别结果（处理被拆分的单词）

#### 4.5.2 音标查询流程

```
OCR 识别出单词 "apple"
    │
    ▼
查询本地词库
    │
    ├── 命中 → 返回音标 /ˈæpl/
    │
    └── 未命中 → 提示用户手动输入音标
                └── 保存到本地扩展词库
```

#### 4.5.3 本地词库

- **基础词库**：预置小学阶段常见单词音标（JSON 格式）
- **扩展词库**：用户手动添加的单词音标，持久化存储
- **词库更新**：支持后续通过 APP 更新推送扩充基础词库

#### 4.5.4 音标字体

- 嵌入 **Doulos SIL** 或 **Charis SIL** 字体文件
- 确保 IPA 特殊符号（如 /æ/, /ʃ/, /θ/, /ð/ 等）正确显示

---

### 4.6 发音系统

#### 4.6.1 TTS 方案

- **框架**：`AVSpeechSynthesizer`
- **默认语音**：
  - 美音：`en-US`
  - 英音：`en-GB`
- **播放控制**：
  - 支持慢速播放（`rate = 0.5`）
  - 支持常速播放（`rate = 0.5 ~ 0.6`，AVSpeech 默认值）

#### 4.6.2 发音交互

- 点击音标标签 → 立即播放单词发音
- 发音过程中标签高亮（边框变蓝或背景变色）
- 支持连续点击，中断当前发音并重新播放

#### 4.6.3 发音设置

- 在设置页可切换英音/美音
- 在设置页可调节发音速度（慢速 / 常速 / 快速）

### 4.7 数据存储

#### 4.7.1 数据模型

```swift
// 课本
Textbook {
    id: UUID
    name: String
    coverImagePath: String
    createdAt: Date
    units: [Unit]
}

// 单元
Unit {
    id: UUID
    name: String
    order: Int
    pages: [Page]
}

// 页面
Page {
    id: UUID
    name: String
    imagePath: String
    order: Int
    annotations: [Annotation]
}

// 标注
Annotation {
    id: UUID
    word: String
    phonetic: String
    normalizedX: Double    // 0.0 ~ 1.0
    normalizedY: Double    // 0.0 ~ 1.0
    normalizedWidth: Double
    normalizedHeight: Double
}
```

#### 4.7.2 存储方案

- **图片**：存储于 APP Sandbox 的 Documents 目录
- **结构化数据**：Core Data 或本地 SQLite
- **词库**：APP Bundle 内嵌 JSON + Documents 目录扩展 JSON

## 5. 非功能需求

### 5.1 性能要求

- APP 冷启动时间 < 2 秒
- OCR 识别单页时间 < 3 秒（iPhone 12 及以上机型）
- 点读页加载时间 < 1 秒
- 点击发音响应时间 < 200ms

### 5.2 兼容性要求

- 支持 iOS 16.0 及以上版本
- 适配 iPhone 和 iPad（优先 iPhone，iPad 为 P2）
- 适配深色模式（P2）

### 5.3 离线要求

- 核心功能（拍照、OCR、音标查询、发音）完全离线可用
- 无需网络连接

### 5.4 安全与隐私

- 所有数据仅存储在本地，不上传云端
- 相册访问遵循 iOS 最小权限原则
- 相机访问仅在拍照时使用

## 6. 界面设计原则

### 6.1 视觉风格

- **整体风格**：简洁、清晰、低干扰
- **主色调**：清新蓝或薄荷绿（教育类 APP 常用色）
- **背景色**：白色或极浅灰色，突出课本内容
- **音标标签**：白色半透明底 + 深色文字，确保在各种课本背景下可读

### 6.2 交互原则

- **核心操作不超过 2 步**：从打开 APP 到听到发音，路径最短化
- **点读模式无干扰**：不显示编辑按钮、菜单等无关元素
- **校对模式反馈明确**：每个操作都有即时视觉反馈

## 8. 成功指标

| 指标 | 目标 |
|------|------|
| 孩子独立使用率 | 每天放学后主动使用 ≥ 5 分钟 |
| 家长校对时间 | 单页平均校对时间 ≤ 2 分钟 |
| OCR 首次可用率 | 自动识别后无需大幅校对的比例 ≥ 70% |
| 发音点击率 | 进入点读页后，至少点击 1 个音标的比例 ≥ 90% |

## 9. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| OCR 识别率不稳定 | 用户体验差，校对负担重 | 优化后处理逻辑；提供一键重识；持续测试不同教材版本 |
| 音标标签重叠 | 页面混乱，难以点击 | 实现基础碰撞检测；支持手动拖拽调整 |
| 课本图片变形 | OCR 坐标偏差 | 引导用户正对拍摄；提供透视校正功能 |
| 词库覆盖不足 | 大量单词无法自动标注 | 优先覆盖主流小学教材词汇；提供便捷的手动补充入口 |
